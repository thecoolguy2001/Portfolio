# VISUALIZER PRO - USER MANUAL

## 1. Introduction

Visualizer is an advanced audio-reactive visual experience that transforms music into dynamic, immersive visual effects. It combines cutting-edge WebGL technology with real-time audio analysis to create stunning visualizations synchronized to your music.

The application features 50+ unique visual effects, multiple playback modes, high-quality video recording capabilities, and a powerful custom effect creator for pro users. Whether you're using it for music enjoyment, performances, or creative expression, Visualizer provides a seamless offline experience with no social features or distractions.

## 6. Custom Effect Creator (Pro)

The Custom Effect Creator allows you to build your own visual effects using JavaScript and WebGL shaders. This powerful feature gives you complete control over geometry, materials, and animations.

### 6.1 Accessing the Creator

Press **C** to open the Custom Effect Menu with two options:
- **Load Effect** - Load an effect from a shared code
- **Create New Effect** - Build a new effect from scratch

### 6.2 Effect Structure

A custom effect consists of four JavaScript functions:

#### 1. Create Function (Required)

Creates the visual mesh and adds it to the scene. This function should:
- Create THREE.js geometry
- Apply materials (ShaderMaterial for advanced effects)
- Set up uniforms for animation
- Add the mesh to both `scene` and `visualElements` array
- Set a unique name for the mesh

#### 2. Animate Initial Function (Optional)

Called continuously when no audio is playing. Use this for idle animations driven by time. The function has access to the mesh through the `visualElements` array.

#### 3. Animate Reactive Function (Optional)

Called continuously during audio playback. Receives three arguments:
- `arguments[0]` - Bass frequency average (0-255)
- `arguments[1]` - Mid frequency average (0-255)
- `arguments[2]` - Treble frequency average (0-255)

Use these values to create audio-reactive animations.

#### 4. Adjust Intensity Function (Optional)

Called when the user changes intensity (keys 1-9). Receives:
- `arguments[0]` - Intensity level (1-9)

Use this to adjust effect parameters based on user preference.

### 6.3 Available Globals

Your custom effect code has access to these global variables:
- `THREE` - Three.js library
- `scene` - The main THREE.Scene
- `camera` - The THREE.PerspectiveCamera
- `renderer` - The THREE.WebGLRenderer
- `visualElements` - Array of all effect meshes
- `audioContext` - Web Audio API context
- `analyser` - AnalyserNode for audio data

### 6.4 Example: Pulsing Sphere Effect

Here's a complete example that creates a sphere that pulses with bass frequencies:

**Create Function:**
```javascript
const geometry = new THREE.SphereGeometry(5, 32, 32);
const material = new THREE.MeshBasicMaterial({
  color: 0x00ff00,
  wireframe: true
});
const sphere = new THREE.Mesh(geometry, material);
sphere.name = 'CustomEffect' + (visualElements.length + 1);
sphere.userData.baseScale = 1.0;
scene.add(sphere);
visualElements.push(sphere);
```

**Animate Initial Function:**
```javascript
const sphere = visualElements.find(e => e.name.startsWith('CustomEffect'));
if (sphere) {
  sphere.rotation.y += 0.01;
}
```

**Animate Reactive Function:**
```javascript
const sphere = visualElements.find(e => e.name.startsWith('CustomEffect'));
if (sphere) {
  const bassValue = arguments[0]; // 0-255
  const scale = 1.0 + (bassValue / 255) * 2;
  sphere.scale.set(scale, scale, scale);
  sphere.rotation.y += 0.02;
}
```

**Adjust Intensity Function:**
```javascript
const sphere = visualElements.find(e => e.name.startsWith('CustomEffect'));
if (sphere) {
  const level = arguments[0]; // 1-9
  sphere.userData.baseScale = level / 5;
}
```

### 6.5 Example: Shader-Based Effect

For advanced users, here's an example using custom shaders:

**Create Function:**
```javascript
const geometry = new THREE.PlaneGeometry(20, 20, 128, 128);
const material = new THREE.ShaderMaterial({
  uniforms: {
    time: { value: 0.0 },
    audioLevel: { value: 0.0 },
    bassLevel: { value: 0.0 },
    midLevel: { value: 0.0 },
    trebleLevel: { value: 0.0 }
  },
  vertexShader: `
    uniform float time;
    uniform float bassLevel;
    varying vec2 vUv;
    void main() {
      vUv = uv;
      vec3 pos = position;
      float wave = sin(pos.x * 2.0 + time) * bassLevel * 0.1;
      pos.z += wave;
      gl_Position = projectionMatrix * modelViewMatrix * vec4(pos, 1.0);
    }
  `,
  fragmentShader: `
    uniform float time;
    uniform float midLevel;
    varying vec2 vUv;
    void main() {
      vec3 color = vec3(vUv.x, vUv.y * midLevel * 0.01, 0.5 + sin(time) * 0.5);
      gl_FragColor = vec4(color, 1.0);
    }
  `
});
const mesh = new THREE.Mesh(geometry, material);
mesh.name = 'CustomEffect' + (visualElements.length + 1);
scene.add(mesh);
visualElements.push(mesh);
```

**Animate Initial Function:**
```javascript
const mesh = visualElements.find(e => e.name.startsWith('CustomEffect'));
if (mesh && mesh.material.uniforms) {
  mesh.material.uniforms.time.value += 0.01;
}
```

**Animate Reactive Function:**
```javascript
const mesh = visualElements.find(e => e.name.startsWith('CustomEffect'));
if (mesh && mesh.material.uniforms) {
  mesh.material.uniforms.bassLevel.value = arguments[0];
  mesh.material.uniforms.midLevel.value = arguments[1];
  mesh.material.uniforms.trebleLevel.value = arguments[2];
  mesh.material.uniforms.time.value += 0.02;
}
```

### 6.6 Sharing Effects

After creating an effect:
1. The shareable code is **automatically copied to your clipboard**
2. The code displays at the **bottom-left of the screen for 60 seconds**
3. Share this code with others
4. Recipients can load it using **"Load Effect"** in the Custom Effect Menu

To view the code again:
- Press **C** while your custom effect is active
- The code will be displayed and copied to clipboard

### 6.7 Advanced Tips

- **ShaderMaterial** is recommended for complex visual effects
- Store effect-specific state in `mesh.userData` object
- Use `mesh.userData.baseEffectIntensity` for intensity scaling
- The `time` uniform should increment continuously for animations
- Audio data ranges: bass/mid/treble are 0-255
- Reference built-in effects in the source code for advanced techniques
- Test your effect at different intensity levels (1-9)
- Ensure proper error handling to prevent crashes

### 6.8 Troubleshooting Custom Effects

**Effect doesn't appear:**
- Verify the mesh is added to both `scene` and `visualElements`
- Check the browser console (F12) for JavaScript errors
- Ensure geometry and material are properly initialized

**Effect doesn't respond to audio:**
- Verify the Animate Reactive function uses `arguments[0]`, `arguments[1]`, `arguments[2]`
- Check that uniforms are being updated correctly
- Ensure the effect is active (visible in visualElements)

**Effect crashes the visualizer:**
- Use try-catch blocks around complex operations
- Avoid infinite loops in animation functions
- Check for null/undefined before accessing object properties

## 7. Draw Mode (Pro)

Draw Mode is an exciting creative feature that lets you paint with visual effects directly on the canvas. Instead of watching a single effect fill the screen, you can draw strokes using any effect as your "brush," creating layered, multi-effect compositions that react to your music.

### 7.1 Accessing Draw Mode

Press **D** to toggle Draw Mode on/off. This feature requires Pro access.

When Draw Mode activates:
- The current effect disappears from the main view
- Your cursor becomes a brush that spawns effect instances
- A message confirms "draw mode: on"

### 7.2 Draw Mode Controls

| Key | Function |
|-----|----------|
| **D** | Toggle Draw Mode on/off |
| **Arrow Left/Right** | Change brush effect (cycles through all available effects) |
| **F** | Toggle Fade Mode (strokes automatically fade away) |
| **Backspace** | Erase the last stroke drawn |
| **1-9** | Adjust intensity of all drawn effects |

**Context-Aware Help:** The help menu (H) adapts to Draw Mode. When you're in Draw Mode and press H, you'll see the additional F and Backspace controls. These controls are hidden from the help menu when not in Draw Mode to keep the interface clean and show only relevant options.

### 7.3 How to Draw

1. **Click and drag** your mouse to draw strokes on the canvas
2. Effect instances spawn along your mouse path as you draw
3. Each stroke remembers which effect brush you used
4. Release the mouse button to complete a stroke
5. Change effects with Arrow keys and draw more strokes to layer different effects

### 7.4 Fade Mode

Press **F** to enable Fade Mode. When active:
- New strokes begin fading after **5 seconds**
- The complete fade takes **10 seconds**
- Strokes shrink and become transparent as they fade
- Fully faded strokes are automatically removed (freeing memory)

Fade Mode is perfect for live performances where you want a constantly evolving canvas without manual cleanup.

### 7.5 Best Practices for Draw Mode

**Creative Tips:**
- **Layer contrasting effects** - Combine geometric effects (Neon Grid) with organic ones (Aurora) for visual interest
- **Use fade mode for performances** - Let old strokes disappear naturally while you create new ones
- **Match brush to music** - Use different effects for different parts of a song (geometric for drops, fluid for breakdowns)
- **Experiment with intensity** - Lower intensity (1-3) for subtle backgrounds, higher (7-9) for impactful accents
- **Draw rhythmically** - Time your strokes to the beat for synchronized visual performances

**Performance Optimization:**
- **Limit active strokes** - Too many simultaneous effects will slow down performance
- **Use Fade Mode** - Automatically cleans up old strokes to maintain smooth framerate
- **Erase unused strokes** - Press Backspace to remove strokes you don't need
- **Lower intensity when layering** - Multiple high-intensity effects compound the GPU load
- **Exit and re-enter** - Pressing D twice clears all strokes if things get sluggish
- **Simple effects for dense drawings** - Some effects (like Particles) are lighter than others (like Quantum Architect)

**Recommended Effect Combinations:**
- Aurora + Neon Grid (organic meets geometric)
- Particles + Waves (layered depth)
- Ocean + Glitch (nature meets digital)
- Blob + Pop Art (playful contrast)

### 7.6 Exiting Draw Mode

Press **D** again to exit Draw Mode. This will:
- Clear all drawn strokes from the canvas
- Restore the normal single-effect view
- Return to standard visualization mode

**Note:** All strokes are cleared when exiting. If you want to preserve your creation, use the **R** key to record a video before exiting Draw Mode.

## 9. Troubleshooting

### 9.1 Recording Issues

**Problem:** Recording doesn't save or produces empty files.

**Solution:** Ensure you've pressed spacebar at least once to start the visualizer before recording. The audio context needs to be initialized before recording can capture data. If recording still fails, check browser compatibility (Chrome/Edge recommended).

### 9.2 Microphone Mode Not Working

**Problem:** No visualization in microphone mode.

**Solution:**
- Check that you've granted microphone permission to your browser
- In Chrome: Click the lock icon in the address bar → allow microphone
- On mobile: Check system settings for browser permissions
- Try closing and reopening the browser

### 9.3 Apple Music Connection

**Problem:** Can't connect to Apple Music.

**Solution:**
- Ensure you have an active Apple Music subscription
- Verify you're signed into iCloud
- Clear browser cookies and try authorizing again
- Try a different browser (Safari works best for Apple Music)

### 9.4 Visual Mode Camera Issues

**Problem:** Camera doesn't activate in visual mode.

**Solution:**
- Grant camera permission to your browser
- Some effects don't support visual mode - try a different effect
- On mobile: Use long press to switch between front/back cameras
- Check that no other app is using the camera

### 9.5 File Upload Issues

**Problem:** Can't upload audio files.

**Solution:**
- Ensure you have Pro access (password entered)
- Check the file format (MP3, WAV, M4A, FLAC supported)
- Try a smaller file size if the browser is struggling
- Clear browser cache and try again

### 9.6 Performance Issues

**Problem:** Visualizer is laggy or slow.

**Solution:**
- Lower the intensity level (press 1 or 2)
- Close other browser tabs
- Disable visual mode (camera integration)
- Use a desktop browser instead of mobile
- Check GPU drivers are up to date
