April 2026

Solution

LE pairs a Raspberry Pi with a Camera Module 3, a small USB microphone, and a small USB speaker for input and output. A locally-downloaded LLM handles day-to-day queries on-device, while a local vision LLM gives LE visual intelligence through the camera. A routing layer escalates more demanding requests to a cloud AI of my choice, giving LE the breadth of cloud assistants without sacrificing the privacy and immediacy of on-device processing.

On-Device LLM

A locally-downloaded language model handles everyday assistant queries with no network round-trip.

Local Vision Model

A separate local visual LLM ingests the Camera Module 3 feed so LE can reason about what it sees.

Cloud Escalation

A routing system sends more demanding multi-step or domain-specific commands to a cloud AI of choice.

Face & Mirror UIs

An expressive face UI gives LE personality; a mirror UI provides at-a-glance ambient information.

April 2026

Visual Design Direction

LE has two display surfaces — an expressive face UI and a mirror UI for ambient information. The visual language has to feel like a single character, not two unrelated screens.

Face has personality, not anatomy

Eyes, mouth, and subtle motion convey state. No realistic features — abstraction lets users project warmth into LE without the uncanny valley.

Mirror is a glance, not a dashboard

Time, weather, calendar, alerts — sized for arm's-length reading. Information density is deliberately low.

Quiet by default

Idle state is a calm face on a black background. LE only animates with intent — no constant motion to demand attention.

Privacy as a visible state

When the camera is on, the face shows it. When the cloud is being used, the face shows that too. Users always see what LE is doing.

April 2026

Problem

Today's home assistants — Alexa, Google Home, Siri — are limited. They live in walled-garden ecosystems, send most queries to the cloud, can't see or reason about their environment, and can't perform agentic multi-step tasks.

I wanted a home assistant that combined local privacy, real visual intelligence, and the ability to act as a true agent — extensible enough that I could keep adding capabilities the off-the-shelf assistants don't offer.

April 2026

Target Users

LE is for people who want a smart-home assistant on their own terms — privacy-conscious, but not allergic to the cloud when it's needed.

PH

The Privacy-First Householder

Wants the convenience of a voice assistant without sending every word to a server

Goals

  • Voice queries that stay on-device
  • Visible, controllable cloud usage
  • Replaces Alexa / Google Home in the kitchen

Pain Points

  • Mainstream assistants stream constantly to the cloud
  • No transparency on what's recorded vs. discarded
TT

The Tinkerer

Wants a hackable assistant they can keep extending

Goals

  • Add custom agentic capabilities over time
  • Pair the assistant with the rest of their home stack
  • Control routing between local and cloud models

Pain Points

  • Mainstream assistants are walled gardens
  • DIY assistants ignore visual input entirely
